Apr 29, 2017 | Hits

最近倒腾了一下Dagger 2，发现还是和Spring里面的依赖注入略有区别，稍微麻烦一丢丢，在这里稍作记录（滑稽）本文使用MVP架构，将一个Retrofit的Service动态注入到Presenter中，权当一个小demo吧

参考文献 Dagger 2: Even sharper, less square

Get start

在build.gradle文件下添加dagger2的依赖

Module

和Dagger 1中相似，Module类用于定义内容的提供者（provider），比如说我们在下面ApplicationModule.java文件里面定义了一个Retrofit对象的provider。

@Provides注解用来标记这个方法是一个provider method，@Singleton注解用来标记这个provider method返回的对象返回的是一个单例。

类似的，你可以提供（provide）其他的东西（并不是所有的，只是提供一些想法）

  • Context
  • System services (e.g. LocationManager)
  • REST service (e.g. Retrofit)
  • Database manager (e.g. Realm)
  • Message passing (e.g. EventBus)
  • Analytics tracker (e.g. Google Analytics)

Component

在Dagger 2中，Component是用来链接Module和需要被注入依赖的target之间的纽带。
